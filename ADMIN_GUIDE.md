# คู่มือการใช้งาน Admin Panel - DR.DEN Website

## 📚 ภาพรวมระบบ

ระบบหลังบ้าน (Admin Panel) ของ DR.DEN Website เป็นระบบจัดการเนื้อหาที่เชื่อมต่อกับหน้าบ้าน (Frontend) โดยตรง ข้อมูลที่เพิ่ม/แก้ไข/ลบในหลังบ้าน จะแสดงผลที่หน้าบ้านทันที

---

## 🔐 การเข้าสู่ระบบ

1. เปิดไฟล์ `admin/index.html` ในเบราว์เซอร์
2. ระบบจะ authenticate อัตโนมัติ (mock authentication)
3. หน้า Dashboard จะแสดงสถิติข้อมูลทั้งหมด

---

## 📊 หน้า Dashboard

แสดงข้อมูลสรุป:
- **จำนวนรีวิว** - รีวิวจากลูกค้าทั้งหมด
- **จำนวนโปรโมชั่น** - โปรโมชั่นที่ active
- **จำนวนบทความ** - บทความทั้งหมด
- **จำนวนบริการ** - บริการที่เปิดให้บริการ

## ✨ ฟีเจอร์ที่ใช้งานได้แล้ว

### 1. รีวิว (Reviews) ✅ พร้อมใช้งาน

**การเพิ่มรีวิวใหม่:**
1. คลิกเมนู "รีวิว" ใน Sidebar
2. คลิกปุ่ม "เพิ่มรีวิว" (สีทอง)
3. กรอกข้อมูล:
   - ชื่อลูกค้า
   - บริการที่ใช้
   - คะแนน (1-5 ดาว)
   - ความคิดเห็น
   - สถานะ (ใช้งาน/ไม่ใช้งาน)
4. คลิก "เพิ่มรีวิว"
5. **ข้อมูลจะแสดงที่หน้า reviews.html ทันที!**

**การแก้ไขรีวิว:**
1. คลิกปุ่ม <i class="fas fa-edit"></i> (สีน้ำเงิน)
2. แก้ไขข้อมูล
3. คลิก "บันทึก"

**การลบรีวิว:**
1. คลิกปุ่ม <i class="fas fa-trash"></i> (สีแดง)
2. ยืนยันการลบ

## 💾 การจัดการข้อมูล (Settings)

### Export ข้อมูล
1. คลิกเมนู "การตั้งค่า"
2. คลิก "Export Data"
3. ไฟล์ JSON จะถูกดาวน์โหลด (ชื่อไฟล์: `drden-backup-YYYY-MM-DD.json`)

### Import ข้อมูล
1. คลิกเมนู "การตั้งค่า"
2. คลิก "Import Data"
3. เลือกไฟล์ JSON ที่ export ไว้
4. ยืนยันการนำเข้า

### Clear ข้อมูลทั้งหมด ⚠️
1. คลิกเมนู "การตั้งค่า"
2. คลิก "Clear All Data"
3. ยืนยัน 2 ครั้ง (เพื่อความปลอดภัย)
4. **ข้อมูลทั้งหมดจะถูกลบถาวร!**

## 🔄 การเชื่อมต่อกับหน้าบ้าน

### ระบบทำงานอย่างไร?

1. **Admin Panel** บันทึกข้อมูลใน LocalStorage (`drden_reviews`, `drden_services`, ฯลฯ)

2. **Data Bridge** (`data-bridge.js`) ดึงข้อมูลจาก LocalStorage และจัดรูปแบบ

3. **Frontend Pages** โหลดข้อมูลจาก Data Bridge และแสดงผล

### ตัวอย่างการทำงาน:

```
Admin Panel
    ↓ บันทึกข้อมูล
LocalStorage (drden_reviews)
    ↓ ดึงข้อมูล
Data Bridge (data-bridge.js)
    ↓ จัดรูปแบบ
Frontend (reviews.html)
    ↓ แสดงผล
หน้าเว็บ
```

### หน้าที่เชื่อมต่อแล้ว:
- ✅ **reviews.html** - แสดงรีวิวจาก LocalStorage
- ⏳ **services.html** - จะเชื่อมต่อเมื่อทำ Form เสร็จ
- ⏳ **promotions.html** - จะเชื่อมต่อเมื่อทำ Form เสร็จ
- ⏳ **articles.html** - จะเชื่อมต่อเมื่อทำ Form เสร็จ
- ⏳ **doctors.html** - จะเชื่อมต่อเมื่อทำ Form เสร็จ
- ⏳ **branches.html** - จะเชื่อมต่อเมื่อทำ Form เสร็จ

## 🔧 สำหรับนักพัฒนา

### โครงสร้างไฟล์:

```
DrdenWebsite/
├── admin/
│   ├── index.html      # Admin UI
│   ├── admin.css       # Admin Styles
│   └── admin.js        # Admin Logic (CRUD)
├── data-bridge.js      # เชื่อมต่อ Admin ↔ Frontend
├── reviews.html        # หน้ารีวิว (เชื่อมต่อแล้ว)
├── reviews.js          # โหลดรีวิวจาก LocalStorage
├── services.html       # หน้าบริการ
├── promotions.html     # หน้าโปรโมชั่น
├── articles.html       # หน้าบทความ
├── doctors.html        # หน้าแพทย์
└── branches.html       # หน้าสาขา
```

### LocalStorage Keys:
- `drden_reviews` - ข้อมูลรีวิว
- `drden_services` - ข้อมูลบริการ
- `drden_promotions` - ข้อมูลโปรโมชั่น
- `drden_articles` - ข้อมูลบทความ
- `drden_doctors` - ข้อมูลแพทย์
- `drden_branches` - ข้อมูลสาขา
- `drden_activities` - Log การทำงาน

### API Reference (DataBridge):

```javascript
// ดึงรีวิวทั้งหมด (เฉพาะ active)
const reviews = DataBridge.getReviews();

// ดึงบริการทั้งหมด
const services = DataBridge.getServices();

// ดึงโปรโมชั่นทั้งหมด (เฉพาะที่ยังไม่หมดเวลา)
const promotions = DataBridge.getPromotions();

// ดึงบทความทั้งหมด
const articles = DataBridge.getArticles();

// ดึงแพทย์ทั้งหมด
const doctors = DataBridge.getDoctors();

// ดึงสาขาทั้งหมด
const branches = DataBridge.getBranches();

// ดึงสถิติทั้งหมด
const stats = DataBridge.getStats();
```

## ⚠️ ข้อควรระวัง

1. **ข้อมูลเก็บใน LocalStorage** - หากลบข้อมูล Browser หรือ Clear Cache ข้อมูลจะหาย
2. **ข้อมูลไม่ Sync ระหว่าง Browser** - แต่ละเบราว์เซอร์มี LocalStorage แยกกัน
3. **ไม่มี User Authentication จริง** - ใครก็เข้าได้หากรู้ URL
4. **ไม่มี Validation รูปภาพ** - ยังไม่รองรับ upload รูป (ใช้ URL แทน)
